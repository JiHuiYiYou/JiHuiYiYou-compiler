## 翻译约定（每个 sprint 都用）

### 类型映射（C → jhyy）

| C 类型 | jhyy 类型 | 说明 |
|---|---|---|
| `int` / `i32` | `i32` | |
| `long long` / `i64` | `i64` | |
| `size_t` | `i64` | jhyy 不区分 size_t，统一 i64 |
| `bool` (`<stdbool.h>`) | `bool` | jhyy 原生 |
| `void*` | `*u8` | 不透明指针 |
| `T*` (typed) | `*T` | |
| `char*` (字符串) | `[*]u8` | 用 slice，NUL 终止 |
| `char[N]` (小缓冲) | `[u8; N]` | 数组 |
| `struct Foo` | `type Foo = struct { ... }` | |
| `union { i32 k; void* p; }` | `i32` + `*u8` 平行 + as cast | tagged union 模式 |
| `enum E { A, B }` | `i32` 常量 | jhyy 无 enum 原生 |
| `const char*` literal | `"..."` | |
| `T**` (array of ptr) | `[*]*T` | |

### 字符串处理

- C `strlen` / `strcmp` / `strncmp` → jhyy 用 `[*]u8` slice + 自己实现（`util.jhyy` sprint 1 提供 `slice_eq`、`slice_len`）
- C `strdup` → `arena_strdup`（v0.6 stage 0 已有）
- C `snprintf` / `fprintf` → FFI `fprintf`（变参 FFI v0.6 未做，先用 `extern fn puts` + 字符串拼）

### Arena 模式

- C `arena_alloc(a, sizeof(T))` → `arena_alloc(a, size as i64)`（**jhyy 无 `size_of::<T>()` 内置**，sprint 2 types.jhyy 翻译时用 ABI sizeof 表硬编码 + match 递归实现 `type_size(*Type) -> i64`）
- C `arena_calloc` → `arena_calloc`（sprint 1 提供）
- 数组 C `T *arr = arena_alloc(a, n * sizeof(T))` → `let arr = arena_alloc(a, (n * size) as i64)`（调用方**自己算字节数**，jhyy 无泛型所以无 `alloc_slice::<T>`）

### 错误传播

- C `bool func(...)` 返回成功失败 → jhyy 同
- C `int func(...)` 0=成功 -1=失败 → jhyy `i32`，约定 0=成功
- C `T* func(...)` NULL=失败 → jhyy `*T`，约定 NULL ptr=失败
- 不引入 `Result<T, E>` 模式（**jhyy 无泛型**，phase-3 sprint 3b 才考虑）

### FFI 声明位置

- 所有 `extern fn` 集中在 `compiler/src0/ffi.jhyy`
- sprint 1 第一步：建 ffi.jhyy，列 libc 必需函数（malloc, free, memcpy, memset, fprintf, fopen, fread, fclose, strlen, strcmp）

### 命名约定

- C `snake_case` → jhyy `snake_case`（不变）
- 翻译模块 = C 文件名（`arena.c` → `arena.jhyy`、`parser.c` → `parser.jhyy` 等）
- 新模块（**不在 C 端**）：`util.jhyy` / `ffi.jhyy` — sprint 1 新建
- FFI 函数名前加 `libc_`（如 `libc_malloc`）— 跟 jhyy 自己函数区分

### C ↔ jhyy 互调（sprint 1 M1 起）

- C 编译器调 jhyy 写的 lib：C 主程序 link jhyy 编译出的 .o + 写 C 头文件声明
- 当前 sprint 1 M1（**降级**）选小范围：jhyy 写的 `_driver.jhyy` import arena/ffi/util + link jhyy_helpers.o 编出 _src0_driver.exe，行为跟 v0.6 arena_test 一致（EXIT=42）— **不是 C 调 jhyy lib**（不可达），是 **jhyy 编 jhyy 基础设施 + C helper 桥**

### 1.2 util.jhyy 设计（**jhyy 版编译器内部工具库**，plan 修订原"抽取型"措辞）

> **核心澄清（plan 修订）**：util.jhyy 不是从 C 端"抽 API"。它是 **jhyy 版编译器内部的工具库**，C 端 codegen.c / symtab.c / sema.c **不重写、不删**，v1.0 全程跟 jhyy 版编译器**没有调用关系**。两个版本并存直到 v2.0+ 才删 C 端。jhyy 版 codegen.jhyy（sprint 4）用 jhyy StringBuilder。

**StringBuilder**（**jhyy 内部使用**，不是替换 C 端 sprintf）：
- jhyy 等价：`type StringBuilder = struct { buf: [*]u8, len: i64, cap: i64 }`
- 关键操作：`sb_init(a)` / `sb_append(a, &sb, src, n)` / `sb_append_cstr(a, &sb, s)` / `sb_appendf(a, &sb, fmt, val)`（单 i32，**内部 libc_sprintf 包装**）/ `sb_cstr(&sb)` / `sb_reset(&sb)` / `sb_free(&sb)`
- arena 分配 buf，扩容时 libc_realloc

**HashMap**（**jhyy 内部使用**，C 端 symtab.c 不动）：
- jhyy 等价：`type HashMap = struct { buckets: [*]*u8, nbuckets: i64, count: i64, arena: *Arena }`
- FNV-1a 64-bit hash + 开放寻址 + 线性探测（参考 v0.2.1 P0-A 修复后的 symtab.c 模式）
- sprint 2 翻译 symtab.jhyy 时直接复用 util.jhyy 的 HashMap（C 端 symtab.c 同时保留）

**Diagnostic**（**jhyy 内部使用**，C 端 sema.c 不动）：
- jhyy 等价：`type Diagnostic = struct { loc: SourceLoc, msg: [*]u8 }` + `fn diag_emit(d: *Diagnostic)` 输出到 stderr（走 jhyy_helpers 的 `jh_print_loc_stderr(loc)` + `jh_fputs_stderr(msg)`）
- `SourceLoc` = `{ filename: [*]u8, line: i32, col: i32 }`
- **变参处理**：jhyy 端构造 Diagnostic 前先用 StringBuilder 拼好完整消息（sema 错误格式：`"type mismatch: expected i32, got f64"`），彻底绕开 JHYY 端 variadic

### 1.3 arena.jhyy 翻译要点

**翻译**（不是抽取）：

### 2.2 ast.jhyy 翻译要点

**Tagged union 模式**（**重大决策**）：
- C 端 `typedef struct Node { NodeKind kind; union { ... } payload; } Node;` + `Node + extra inline` 模式（`new_node(extra)` 把 payload 紧跟 Node 一次 alloc）
- **jhyy 不能 1:1 翻译 inline payload** —— jhyy 无 `sizeof(Node)` 内置，alloc 跟 payload cast 难
- jhyy 等价：`type Node = struct { kind: i32, loc: SourceLoc, type: *Type, payload: *u8 }`（payload 是 untyped 指针，访问时 `as *NodeBinary`）
- 49 个 payload struct（`NodeBinary` / `NodeCall` / `NodeIf` / ...）每个独立定义
- 49 个 constructor (`fn new_node_binary(a, op, lhs, rhs) -> *Node`) + 49 个 accessor (`fn node_binary_data(n) -> *NodeBinary`) = **98 个 fn**（这是 ast.jhyy 行数膨胀的主因，C 用 macro 展开）
- 构造一次 Node：先 `arena_alloc(a, sizeof_Node)` → 填 kind/loc/type，再 `arena_alloc(a, sizeof_NodeBinary)` → 填 payload，再 `node.payload = payload_ptr as *u8`。**两次 alloc** vs C 一次

**避开 § 0 codegen 坑**：
- ast.jhyy 大量 `let mut` (counter / loop) → 命中 #2 let mut bug，每次 `let` 都要想清楚
- Node 含 4 字段 (kind/loc/type/payload) 嵌套, payload 指针是 untyped → **不**命中 #5（Node 平铺, 不嵌套 struct）
- symtab.jhyy 哈希 key 是 `*u8`（sym 名字）→ 命中 #6 qbe_type_of(i8)（hash 算 char by char），**直接复用 util.jhyy hash_string 的 `*i32 + shift+mask` workaround**（不重新发明）

**构造函数**：
- C `static Node *new_node(Arena *a, NodeKind k, SourceLoc loc, size_t extra)` → jhyy `fn new_node(a: *Arena, k: i32, loc: SourceLoc) -> *Node`（不传 extra, payload 单独 alloc）

### 2.3 types.jhyy 翻译要点

**递归类型**：
- C `struct Type { TypeKind kind; union { ... } u; } Type;`（union 内有 `Type *elem`、`Type **params` 等指针递归）
- jhyy 等价：`type Type = struct { kind: i32, ... }` + 所有递归字段用 `*Type`
- 关键 API：`type_primitive` / `type_pointer` / `type_array` / `type_struct` / ... / `type_size` / `type_align` / `type_eq` / `type_to_string`
- **jhyy 端 size / align / eq / to_string 都是 match 递归**（C 端 switch 改写）

**prims_inited 全局缓存问题（sprint 2 落, sprint 5 用）**：
- C 端 `static Type prim_i32, ..., prim_bool; static int prims_inited = 0; static void init_primitives(void) { ... }` —— 全局静态缓存 primitive types
- **jhyy 无全局变量**（v0.6 阶段）。三种解决：
  - **A (推荐)**: jhyy 端用 `let mut prims: [Type; 11] = [Type { kind: KIND_VOID, prim: 0 }; 11]; let prims_inited: i32 = 0;` 全局栈变量, 但**jhyy 也没真正的全局变量**, 只能 `static` 数组（如果 v0.6 支持）
  - **B**: 在 jhyy 端每个 Type * 单独 arena_alloc, 不缓存。**性能差一点**（每次 type_primitive 都 alloc）, 但简单
  - **C**: **sprint 2 决定**: 缓存放在 compiler top-level scope 的 arena, 入口是 `fn type_init(a: *Arena) -> *TypeArena` 返回 *TypeArena（带 11 个 primitive 指针 + 后续 alloc 的 type）, main.jhyy 启动时调一次, 后续函数传 *TypeArena 进去
- **实施: 选 C 方案**（最干净, 跟 jhyy 函数式风格一致, 性能最好）。`type_primitive` 签名改为 `fn type_primitive(ta: *TypeArena, prim: i32) -> *Type`, 内部 lazy init primitives
- **sprint 5 main.jhyy 调 `type_init(arena)`, 把 *TypeArena 传遍所有 types 调用**

**type_size / type_align 实现**（**v0.6.2 patch 已修 qbe_type_of, type_align 函数本身没变, 这里只是确认**）：
- **jhyy 无 `size_of::<T>()` 内置**（v0.6.2 仍无，jhyy 不内建）
- 走 match 递归（**对齐 C 端 switch 行为**）
- payload 构造用 `arena_alloc` + field-by-field 设置

**type_size / type_align match 实现**（参考 C 端 types.c:121 / 147）：

```jhyy
// 基础类型硬编码（来自 docs/abis/jhyy-abi-v1.0.0.md § 类型布局）
fn primitive_size(kind: i32) -> i64 {
    match kind {
        PRIM_I8 | PRIM_I16 | PRIM_I32 | PRIM_U8 | PRIM_U16 | PRIM_U32 | PRIM_BOOL => 4,
        PRIM_I64 | PRIM_U64 => 8,
        PRIM_F32 => 4,
        PRIM_F64 => 8,
        _ => 8
    }
}

// 复合类型递归
fn type_size(t: *Type) -> i64 {
    match t.kind {
        KIND_PRIMITIVE => primitive_size(t.prim),
        KIND_POINTER | KIND_FUNC => 8,
        KIND_SLICE => 16,            // {ptr, len}
        KIND_ARRAY => t.array_count * type_size(t.array_elem),
        KIND_STRUCT => t.struct_size,  // types.jhyy 在 type_struct 构造时存好
        KIND_ENUM => 8,                // tag (4) + payload, 简化 8
        KIND_VOID => 0,
        _ => 8
    }
}
```

参考 `docs/abis/jhyy-abi-v1.0.0.md` 完整 sizeof 表（行 33-176 含 struct 偏移 / 切片 / 数组 / 函数指针 / Unit 类型）。

**type_align 同理**：原始类型 alignof == sizeof（bool 除外，bool=1），切片 16 = max(ptr, len)，struct = max(field align)。

## Sprint 4: 后端 (ir + codegen)

### 4.1 翻译目标

| C 文件 | jhyy 文件 | 复杂度 |
|---|---|---|
| `compiler/src/ir.h/c` (282 行) | `compiler/src0/ir.jhyy` | 中（IRBuf + emit_* 系列） |
| `compiler/src/codegen.h/c` (**1552 行，phase-2 最大单模块**) | `compiler/src0/codegen.jhyy` | 高（AST → QBE IL，拆 3 commit） |

### 4.2 ir.jhyy 翻译要点

- C `IRBuf { Arena *arena; char *buf; size_t len, cap; int next_tmp, next_block; }` → jhyy 等价 `type IRBuf = struct { arena: *Arena, buf: [*]u8, len: i64, cap: i64, next_tmp: i32, next_block: i32 }`
- `ir_emit` 用变参 — jhyy 无变参，**v1.0 拆成多个固定参数版本**（`ir_emit1` / `ir_emit2` / ...）或用 `StringBuilder` + `sb_appendf` 替代
- `ir_emit_ret` / `ir_emit_jnz` / `ir_emit_phi` 保持单 emit 形式

### 4.3 codegen.jhyy 翻译要点

- AST → QBE IL 是 phase-2 中**翻译 / 验证比最差**的模块（C 版就有大量特殊处理）
- 关键 oracle：**给定 AST，jhyy 版 codegen 输出 QBE IL 跟 C 版 byte-equal** — 这是 sprint 4 主要验证手段
- sprint 4 末做"核心场景 diff"：选 5-10 个 .jhyy 测试，diff C 版 codegen 跟 jhyy 版 codegen 输出的 .il

**struct pass-by-value sret 处理**：
- v0.4 ABI 锁定：sret pointer 作为第一个隐藏参数
- v0.6 已支持
- 翻译时按 C 版的 sret emit 模式 1:1

### 4.4 QBE IL 写盘（**关键风险**）

按 memory `feedback_qbe_crlf_root_cause`：jhyy 端写 QBE IL 必须 `fopen("wb")`，否则 Windows MSVCRT 把 `\n` 转 `\r\n` 污染 IL。

```jhyy
// 错误 ❌
let fp = libc_fopen(path, "w" as [*]u8);

// 正确 ✅
let fp = libc_fopen(path, "wb" as [*]u8);
```

**sprint 4 第一周**审查所有 fopen 调用，确保都用了 "wb"（jhyy 端）。这个 bug v0.6 修过 C 端（main.c:85, 418），sprint 4 翻译到 jhyy 端必须重做。

### 4.5 风险

- **QBE IL 写盘 binary mode** — sprint 4 必做正确
- codegen 字符串拼接：StringBuilder 已用（sprint 1）
- 大模块：**codegen.c 1552 行（phase-2 最大单模块，比 parser.c 982 行还大 58%）**，sprint 4 拆 3 个子 commit（codegen_expr → codegen_stmt → codegen_decl/ty），每 commit 末跑 regress，翻译工作量 ~2500 行 jhyy
- **变异构 QBE IL diff（plan 修订）**：byte-equal 硬指标**不接受降级**。sprint 4 翻译 codegen.jhyy 时**强制保证** codegen_expr / codegen_stmt / codegen_decl 各子模块 emit 顺序一致 + tmp 编号单调（C 版 ir_next_tmp 单调递增是天然的，jhyy 版保持同样顺序）。空白差异（多空行/缩进）通过共用 helper emit（如 `ir_emit_str(ir, "  ")`）保持一致。**不允许出现"接受微小差异"**。如实施发现差异无法消除，**回滚 sprint 4 重新设计**，不降级完成定义。

### 4.6 验证

1. `jhyy (C 版) 编译 src0/{ir,codegen}.jhyy` 全成功
2. ir 单元测试：emit 各指令 + 临时变量命名
3. codegen 单元测试：每个 AST 节点 → QBE IL（INT / BINARY / CALL / IF / LET / MATCH / ...）
4. **关键 oracle**: 5-10 个 .jhyy 测试，C 版 codegen 输出 .il 跟 jhyy 版输出 .il diff（**目标 byte-equal，硬指标不接受微小差异降级**）
5. **M4 局部**: C 编译器走完整前端+后端 — 输入 .jhyy 源 → 输出 QBE IL → QBE 工具 → .s → gcc → .exe；regress.py 43/46 全过

### 5.3 Stage 1 验证脚本

```bash
# compiler/tests/bootstrap/stage1.sh
set -e

JHYY_C=./compiler/build/bin/jhyy            # C 版 (sprint 0)

  $JHYY_0 build $test -o /tmp/test_0.il
  $JHYY_1 build $test -o /tmp/test_1.il
```
