## Sprint 6A: 切片 codegen

切片 `[*]T` 在 v0.5.0 已通过 sema 接受（`type_size = 16`、`type_align = 8`），但 codegen 完全缺失，无法在表达式上下文使用。本 sprint 补齐 4 个用法：字面量、索引、子切片、`len` 内置。

### 6A.1 类型表示（已在 types.c 完成）

**现状**：types.c:137 已经返回 16 bytes (`case KIND_SLICE: return 16; /* ptr + len */`)。

**问题**：16 字节是 `*T` (8) + `i64` (8)。这是按 QBE struct layout 写的（恰好对应 `{l, l}`），但**实际代码注释说 24 字节是错的**——那是参考方案里的笔误。我们采用 16 字节。

#### Files: `types.c` (lines 137, 152)

```c
case KIND_SLICE:   return 16; /* { *T, i64 } */
...
case KIND_SLICE:     return 8; /* max alignment of (ptr, i64) */
```

**不变**——已经是 16/8。

#### ABI 决策（关键）

Windows x64 PE 上 16 字节 struct 走 **寄存器返回**（RCX + RDX 两个寄存器），无需 sret。这意味着切片返回/传递都按 value 走 2 个寄存器。

但调用方语法 (`func returning slice` / `slice in args`) 还没支持——v0.6.0 **只支持 local slice 变量、slice 索引、slice 子切片、slice → ptr decay**。返回 slice 不在 v0.6 范围。

### 6A.2 切片字面量

#### Problem

JHYY 需要支持：
```jhyy
let s: [*i32] = &[1, 2, 3];
```

数组字面量 `[1, 2, 3]` 是 `[i32; 3]`，需先 decay 成 `[*i32]` slice。

#### Root Cause

v0.3.0 数组 codegen 后，数组值是栈分配的固定大小 buffer，没有"头 + 长度"结构。切片字面量需要把数组的"栈地址 + 元素数"打包成 `{ptr, len}` 结构。

#### Files: `parser.c`, `sema.c`, `codegen.c`

**Step 1**: parser 识别 `&[expr, expr, ...]` 前缀。

**在 parser.c 的 `parse_expr_primary` (~line 380) 末尾**:

```c
if (lexer_peek(&parser->lexer) == TOKEN_AMP &&
    lexer_peek_at(&parser->lexer, 1) == TOKEN_LBRACK) {
    /* & [expr, expr, ...]  →  array literal that decays to slice */
    lexer_next(&parser->lexer);  /* consume & */
    lexer_next(&parser->lexer);  /* consume [ */
    Node **items = arena_alloc(arena, 8 * sizeof(Node *));
    size_t nitems = 0;
    while (lexer_peek(&parser->lexer) != TOKEN_RBRACK) {
        items[nitems++] = parse_expr(parser);
        if (lexer_peek(&parser->lexer) == TOKEN_COMMA)
            lexer_next(&parser->lexer);
    }
    expect(lexer, TOKEN_RBRACK);
    Node *arr = ast_new_array_lit(arena, loc, items, nitems);
    /* wrap as slice literal — type is [*]elem */
    return ast_new_slice_lit(arena, loc, arr);
}
```

**Step 2**: ast.h/c 新增节点：

```c
typedef enum {
    ...
    NODE_SLICE_LIT,   /* & [expr, ...] */
    ...
} NodeKind;

typedef struct NodeSliceLit {
    Node *array;  /* underlying NODE_ARRAY_LIT */
} NodeSliceLit;

Node *ast_new_slice_lit(Arena *a, SourceLoc loc, Node *array);
```

**Step 3**: sema.c 处理 NODE_SLICE_LIT：取出 array 的元素类型 `T`，构造 `[*]T` 作为表达式类型：

```c
case NODE_SLICE_LIT: {
    Type *arr_t = sema_infer_type(sema, d->array);
    if (!arr_t || arr_t->kind != KIND_ARRAY) {
        // error: slice literal must wrap an array literal
    }
    Type *elem = arr_t->array.elem;
    n->type = type_slice(sema->arena, elem);
    break;
}
```

**Step 4**: codegen.c 处理 NODE_SLICE_LIT：先 codegen array（产生栈 alloc 16 字节 + store 三个元素），然后取 alloc 出来的 base address，构造 `{ptr, len}` 结构：

```c
case NODE_SLICE_LIT: {
    NodeArrayLit *ad = node_array_lit_data(d->array);
    Type *elem = n->type->slice.elem;
    int esz = (int)type_size(elem);
    if (esz < 4) esz = 4;
    int total = esz * (int)ad->nitems;
    if (total < 4) total = 4;  /* QBE alloc alignment constraint */

    /* Allocate array storage */
    IRVal arr_slot = ir_new_tmp(cg->ir, 'l');
    ir_emit_alloc(cg->ir, arr_slot, total);
    /* Store elements (reuse array codegen) */
    for (size_t i = 0; i < ad->nitems; i++) {
        IRVal idx = ir_new_const_int(cg->ir, (int)i);
        IRVal val = cg_expr(cg, ad->items[i]);
        IRVal off = ir_new_tmp(cg->ir, 'w');
        ir_emit(cg->ir, "    %%t%d =w mul %%t%d, %d\n", off.id, idx.id, esz);
        IRVal addr = ir_new_tmp(cg->ir, 'l');
        ir_emit(cg->ir, "    %%t%d =l add %%t%d, %%t%d\n",
                addr.id, arr_slot.id, off.id);
        /* Use type-aware store */
        cg_emit_store(cg, addr, val, elem);
    }

    /* Construct slice {ptr, len} — stack alloc 16 bytes */
    IRVal slice_slot = ir_new_tmp(cg->ir, 'l');
    ir_emit_alloc(cg->ir, slice_slot, 16);
    /* field 0: ptr = arr_slot */
    ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", arr_slot.id, slice_slot.id);
    /* field 8: len = nitems */
    IRVal len_val = ir_new_const_int(cg->ir, (int)ad->nitems);
    IRVal len_off = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l add %%t%d, 8\n", len_off.id, slice_slot.id);
    ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", len_val.id, len_off.id);
    return slice_slot;
}
```

> **注意**：`ir_new_const_int` 已在 `ir.h` 定义。slice 是 16 字节 struct，slot 地址就是表达式结果（调用方读 `.ptr` 用 loadl，读 `.len` 用 `loadl at slot+8`）。

### 6A.3 索引与字段访问

#### Problem

切片索引 `s[i]` 已经在数组索引里有逻辑（`cg_index`），但 slice 没有 base address — 它本身就是 `{ptr, len}` 结构。要先读 `ptr` 字段，再加偏移。

同时切片需要 `.len` 字段访问（不像数组是固定大小，slice 是动态的）。

#### Files: `codegen.c`

**Step 1**: `cg_index` 在 NODE_IDENT 解析目标时，需要先看 `target->type->kind`：

```c
case NODE_INDEX: {
    NodeIndex *d = node_index_data(n);
    IRVal idx = cg_expr(cg, d->index);

    /* Determine element type from the array/slice */
    Type *arr_t = d->array->type;
    IRVal base;
    int esz;
    if (arr_t->kind == KIND_ARRAY) {
        base = cg_expr(cg, d->array);  /* array value IS the pointer */
        esz = (int)type_size(arr_t->array.elem);
    } else if (arr_t->kind == KIND_SLICE) {
        /* Slice: base is at *base (first field) */
        IRVal slice_addr = cg_expr(cg, d->array);
        base = ir_new_tmp(cg->ir, 'l');
        ir_emit(cg->ir, "    %%t%d =l loadl %%t%d\n", base.id, slice_addr.id);
        esz = (int)type_size(arr_t->slice.elem);
    } else {
        // error
        break;
    }
    if (esz < 4) esz = 4;

    IRVal off = ir_new_tmp(cg->ir, 'w');
    ir_emit(cg->ir, "    %%t%d =w mul %%t%d, %d\n", off.id, idx.id, esz);
    IRVal addr = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l add %%t%d, %%t%d\n",
            addr.id, base.id, off.id);

    IRVal val = ir_new_tmp(cg->ir, qbe_type_of(n->type));
    cg_emit_load(cg, val, addr, n->type);
    return val;
}
```

**Step 2**: `NODE_FIELD` 访问 `.len` (slice 字段):

```c
case NODE_FIELD: {
    NodeField *d = node_field_data(n);
    Type *st = d->target->type;
    if (st->kind == KIND_SLICE) {
        /* .ptr = offset 0, .len = offset 8 */
        IRVal slice_addr = cg_expr(cg, d->target);
        if (strcmp(d->field->name, "ptr") == 0) {
            IRVal v = ir_new_tmp(cg->ir, 'l');
            ir_emit(cg->ir, "    %%t%d =l loadl %%t%d\n", v.id, slice_addr.id);
            return v;
        }
        if (strcmp(d->field->name, "len") == 0) {
            IRVal off = ir_new_tmp(cg->ir, 'l');
            ir_emit(cg->ir, "    %%t%d =l add %%t%d, 8\n", off.id, slice_addr.id);
            IRVal v = ir_new_tmp(cg->ir, 'l');
            ir_emit(cg->ir, "    %%t%d =l loadl %%t%d\n", v.id, off.id);
            return v;
        }
    }
    /* existing struct field logic ... */
}
```

### 6A.4 子切片 `s[start..end]`

#### Problem

```jhyy
let sub: [*i32] = s[1..3];   // {s.ptr+1, 2}
```

需要解析 `s[start..end]` 二元语法（已经支持 `for i in 0..N`），但作为表达式结果是 `{ptr+off, len=end-start}`。

#### Files: `parser.c`, `ast.h/c`, `sema.c`, `codegen.c`

**Step 1**: parser.c 解析 `IDENT [ EXPR .. EXPR ]`：

```c
/* In parse_postfix (Pratt), after parsing `IDENT` then `[expr]`,
   check if next token is `..` followed by `]` */
if (lexer_peek(&parser->lexer) == TOKEN_DOTDOT) {
    /* slice subrange */
    lexer_next(&parser->lexer);  /* consume .. */
    Node *end = parse_expr(parser);
    expect(lexer, TOKEN_RBRACK);
    return ast_new_slice_range(arena, loc, base, start_idx, end);
}
```

但 `for i in 0..N` 也用 `..`，需要避免歧义。**更简单**：子切片用单独语法 `IDENT[start..end]`，在 `[expr]` 紧接 `..` 时识别。Parser 里 `parse_index_or_slice` 替换 `parse_index`：

```c
Node *parse_index_or_slice(Parser *p, Node *base) {
    expect(&p->lexer, TOKEN_LBRACK);
    Node *start = parse_expr(p);
    if (lexer_peek(&p->lexer) == TOKEN_DOTDOT) {
        lexer_next(&p->lexer);
        Node *end = parse_expr(p);
        expect(&p->lexer, TOKEN_RBRACK);
        return ast_new_slice_range(p->arena, base->loc, base, start, end);
    }
    expect(&p->lexer, TOKEN_RBRACK);
    return ast_new_index(p->arena, base->loc, base, start);
}
```

**Step 2**: ast 新增 NODE_SLICE_RANGE：

```c
typedef struct NodeSliceRange {
    Node *base;
    Node *start;
    Node *end;
} NodeSliceRange;
```

**Step 3**: sema：要求 base 是 `[*]T` 或 `[T; N]`，结果类型是 `[*]T`：

```c
case NODE_SLICE_RANGE: {
    Type *bt = sema_infer_type(sema, d->base);
    if (bt->kind != KIND_SLICE && bt->kind != KIND_ARRAY) {
        sema_error(sema, "sub-range requires slice or array");
    }
    Type *elem = (bt->kind == KIND_SLICE) ? bt->slice.elem : bt->array.elem;
    n->type = type_slice(sema->arena, elem);
    break;
}
```

**Step 4**: codegen：

```c
case NODE_SLICE_RANGE: {
    Type *bt = d->base->type;
    int esz = (int)type_size(bt->kind == KIND_SLICE ?
                              bt->slice.elem : bt->array.elem);
    if (esz < 4) esz = 4;

    /* base ptr */
    IRVal base;
    if (bt->kind == KIND_SLICE) {
        IRVal slice_addr = cg_expr(cg, d->base);
        base = ir_new_tmp(cg->ir, 'l');
        ir_emit(cg->ir, "    %%t%d =l loadl %%t%d\n", base.id, slice_addr.id);
    } else {
        base = cg_expr(cg, d->base);
    }

    /* start offset = start * esz */
    IRVal start_v = cg_expr(cg, d->start);
    IRVal start_off = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l mul %%t%d, %d\n",
            start_off.id, start_v.id, esz);
    IRVal new_ptr = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l add %%t%d, %%t%d\n",
            new_ptr.id, base.id, start_off.id);

    /* new_len = end - start */
    IRVal end_v = cg_expr(cg, d->end);
    IRVal new_len = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l sub %%t%d, %%t%d\n",
            new_len.id, end_v.id, start_v.id);

    /* Pack into new slice struct */
    IRVal slot = ir_new_tmp(cg->ir, 'l');
    ir_emit_alloc(cg->ir, slot, 16);
    ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", new_ptr.id, slot.id);
    IRVal off8 = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l add %%t%d, 8\n", off8.id, slot.id);
    ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", new_len.id, off8.id);
    return slot;
}
```

### 6A.5 `len(s)` 内置函数

#### Problem

```jhyy
fn len(s: [*i32]) -> i64 {
    s.len
}
```

需要让用户能写 `len(s)` 拿到 slice 长度。两种实现：

**方案 A**：让 `len` 是 keyword + 内置 codegen path（特殊语法）。
**方案 B**：让 `len` 是普通标识符，semantically 解析为 `s.len` 字段访问。

**采用方案 B**——更简洁，复用 6A.3 的 `.len` 字段逻辑。

#### Files: `sema.c`

sema 看到 `len(x)` 调用时，检查 x 是 slice 或 array，重写为字段访问：

```c
case NODE_CALL: {
    if (d->callee->kind == NODE_IDENT &&
        strcmp(node_ident_data(d->callee)->sym->name, "len") == 0 &&
        d->nargs == 1) {
        Node *arg = d->args[0];
        Type *t = sema_infer_type(sema, arg);
        if (t->kind == KIND_SLICE) {
            /* rewrite len(x) → NODE_FIELD{arg, .len} */
            Sym *len_sym = symtab_lookup(sema->globals, "len");
            n->kind = NODE_FIELD;
            n->field.target = arg;
            n->field.field = sym_field("len", KIND_PRIMITIVE, PRIM_I64);  /* synthetic */
            n->type = type_primitive(sema->arena, PRIM_I64);
            break;
        }
        if (t->kind == KIND_ARRAY) {
            /* len([T;N]) = N (constant) */
            n->kind = NODE_INT_LIT;
            n->int_lit.value = (int)t->array.count;
            n->type = type_primitive(sema->arena, PRIM_I64);
            break;
        }
    }
    /* existing call check ... */
}
```

### 6A.6 数组 → 切片衰减（显式 cast）

#### Problem

```jhyy
let arr: [i32; 3] = [1, 2, 3];
let s: [*i32] = arr as [*i32];
```

JHYY 不引入隐式 decay（避免歧义），但 `as` cast 应该支持数组 → 切片。

#### Files: `sema.c` (`cast` 处理) + `codegen.c`

sema：增加 case：

```c
case KIND_ARRAY:
    if (target->kind == KIND_SLICE &&
        type_eq(t->array.elem, target->slice.elem)) {
        /* ok: array → slice cast (no-op at IR level,
           but semantically creates a slice value from the array) */
        n->type = target;
        break;
    }
    /* fallthrough to error */
```

codegen：NODE_CAST 处理数组 → 切片时，构造 slice struct：

```c
case NODE_CAST: {
    Type *from = n->cast.expr->type;
    Type *to = n->type;
    if (from->kind == KIND_ARRAY && to->kind == KIND_SLICE) {
        IRVal arr_base = cg_expr(cg, n->cast.expr);
        int nitems = (int)from->array.count;
        IRVal slot = ir_new_tmp(cg->ir, 'l');
        ir_emit_alloc(cg->ir, slot, 16);
        ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", arr_base.id, slot.id);
        IRVal off8 = ir_new_tmp(cg->ir, 'l');
        ir_emit(cg->ir, "    %%t%d =l add %%t%d, 8\n", off8.id, slot.id);
        IRVal len_v = ir_new_const_int(cg->ir, nitems);
        ir_emit(cg->ir, "    storel %%t%d, %%t%d\n", len_v.id, off8.id);
        return slot;
    }
    /* existing cast logic (int↔float, ptr↔int, etc.) */
}
```

## Sprint 6B: 模块命名空间

当前 `import math;` 后 `math.jhyy` 的 `factorial` 会被扁平化到全局作用域（v0.4 实现）。多文件模块同名时冲突。本 sprint 引入 `math::factorial()` 限定语法，并对全局符号做 mangling。

### 6B.1 符号 mangling 规则

| 场景 | v0.5 (现状) | v0.6 后 |
|------|------------|---------|
| `math.jhyy::factorial` 编译为 | `$factorial` | `$math__factorial` |
| 调用 `math::factorial(5)` 编译为 | (不识别) | `$math__factorial(5)` |
| 调用 `factorial(5)` 编译为 | `$factorial(5)` | (错误: 未在当前模块定义) |
| `extern fn printf` 编译为 | `$printf` | `$printf` (不变) |

**双下划线 `__` 分隔** (避开单 `_`，单下划线在标识符里合法)。

### 6B.2 语法：限定调用

```jhyy
import math;

fn main_jhyy() -> i32 {
    math::factorial(5)        // ✅ 限定调用 → $math__factorial
    // factorial(5)           // ❌ sema 错误：factorial not in current module
}
```

**实现**：在 Pratt parser 的 postfix 阶段检测 `IDENT :: IDENT ( ... )`。

### 6B.3 解析与 AST 节点

#### Files: `parser.c`, `ast.h/c`, `symtab.h/c`, `sema.c`, `codegen.c`, `main.c`

## Sprint 6C: `as` 类型转换补全

v0.5 已支持整数 ↔ 浮点 ↔ 浮点。v0.6 加 `*T ↔ usize` 和 `i64 ↔ usize`，用于自举的 hash 计算和 ptr arithmetic。

### 6C.1 新增转换矩阵

| From | To | 实现 |
|------|----|-----|
| `*T` | `usize` | QBE 原生 `loadl` (zero-extend) |
| `usize` | `*T` | QBE 原生 `storel` (int → ptr) |
| `i32` | `usize` | 同 size，无操作 (但 codegen 仍 emit `cast`) |
| `i64` | `usize` | QBE 都是 `l`，bitcast 等价 |
| `usize` | `i64` | 同上 |

### 6C.2 sema.c

#### Files: `sema.c` (`infer_type` `NODE_CAST` case)

```c
case NODE_CAST: {
    Type *from = n->cast.expr->type;
    Type *to = n->type;
    /* existing cases ... */

    /* ptr → usize */
    if (from->kind == KIND_POINTER &&
        to->kind == KIND_PRIMITIVE && to->prim == PRIM_U64) {
        /* ok */
        break;
    }
    /* usize → ptr */
    if (from->kind == KIND_PRIMITIVE && from->prim == PRIM_U64 &&
        to->kind == KIND_POINTER) {
        /* ok */
        break;
    }
    /* i32/i64 ↔ usize (both word-sized or both long-sized) */
    if (from->kind == KIND_PRIMITIVE && to->kind == KIND_PRIMITIVE) {
        int fs = type_size(from), ts = type_size(to);
        if (fs == ts) {
            /* no-op cast */
            break;
        }
        /* different sizes → existing exts/truncd logic */
    }
    sema_error(sema, "invalid cast from %s to %s",
               type_to_string(from), type_to_string(to));
    break;
}
```

### 6C.3 codegen.c

#### Files: `codegen.c` (`cg_cast` 函数)

```c
case NODE_CAST: {
    Type *from = n->cast.expr->type;
    Type *to = n->type;
    IRVal v = cg_expr(cg, n->cast.expr);

    /* ptr → usize: same QBE type 'l', just return v */
    if (from->kind == KIND_POINTER &&
        to->kind == KIND_PRIMITIVE && to->prim == PRIM_U64) {
        return v;  /* already 'l' */
    }
    /* usize → ptr: same 'l' */
    if (from->kind == KIND_PRIMITIVE && from->prim == PRIM_U64 &&
        to->kind == KIND_POINTER) {
        return v;
    }
    /* i32 ↔ usize: QBE type 'w' ↔ 'l', use exts/casti */
    if (from->kind == KIND_PRIMITIVE && to->kind == KIND_PRIMITIVE) {
        char fq = qbe_type_of(from), tq = qbe_type_of(to);
        if (fq == tq) return v;
        if (type_size(from) < type_size(to)) {
            IRVal r = ir_new_tmp(cg->ir, tq);
            if (from->prim == PRIM_U8 || from->prim == PRIM_U16 ||
                from->prim == PRIM_U32) {
                ir_emit(cg->ir, "    %%t%d =%c extu %%t%d\n",
                        r.id, tq, v.id);
            } else {
                ir_emit(cg->ir, "    %%t%d =%c exts %%t%d\n",
                        r.id, tq, v.id);
            }
            return r;
        } else {
            IRVal r = ir_new_tmp(cg->ir, tq);
            ir_emit(cg->ir, "    %%t%d =%c cast %%t%d\n",
                    r.id, tq, v.id);
            return r;
        }
    }
    /* existing int↔float logic ... */
}
```

### 6D.1 import_test.jhyy 路径 bug

#### Problem

`import_test.jhyy` 在 `compiler/tests/examples/` 下，`import mylib;` 解析时 `resolve_imports` 用 `main_path` (即 `compiler/tests/examples/import_test.jhyy`) 的目录作为 import base。

#### Root Cause

#### Files: `main.c` (`resolve_one_import` line 96-100)

**修复**：import 解析 fallback 链：

1. main_path 目录（`compiler/tests/examples/`）
2. cwd（如果 main_path 找不到）
3. 报错

```c
static int try_read_import(const char *path, char **out_buf) {
    *out_buf = read_file(path);
    return *out_buf ? 0 : -1;
}

static char *resolve_import_path(const char *mod_dir, const char *mod_name,
                                  const char *cwd) {
    char path[512];
    /* Try 1: mod_dir */
    snprintf(path, sizeof(path), "%s/%s.jhyy", mod_dir, mod_name);
    if (file_exists(path)) return strdup(path);
    /* Try 2: cwd */
    if (cwd && strcmp(cwd, mod_dir) != 0) {
        snprintf(path, sizeof(path), "%s/%s.jhyy", cwd, mod_name);
        if (file_exists(path)) return strdup(path);
    }
    return NULL;
}
```

`file_exists` 用 `access(path, 0) == 0` 实现。

/* arena.c 原版 */
typedef struct Arena {
    char *base;
    char *cursor;
    char *end;
} Arena;

void arena_init(Arena *a, size_t default_size) {
    a->base = NULL; a->cursor = NULL; a->end = NULL;

void *arena_alloc(Arena *a, size_t size) {
    /* bump allocation */
    char *p = a->cursor;
    a->cursor += size;
    return p;
}

void arena_reset(Arena *a) {

}

void arena_free(Arena *a) {

// 等价于 compiler/src/arena.c (简化版)

extern fn malloc(size: i64) -> *u8;
extern fn free(p: *u8);

type Arena = struct {
    base: *u8,
    cursor: *u8,
    end: *u8,
};

fn arena_init(a: *Arena, default_size: i64) {
    a.base = malloc(default_size);
    a.cursor = a.base;
    a.end = a.base + default_size;
}

fn arena_alloc(a: *Arena, size: i64) -> *u8 {
    let p: *u8 = a.cursor;
    a.cursor = a.cursor + size;
    p
}

fn arena_reset(a: *Arena) {
    a.cursor = a.base;
}

fn arena_free(a: *Arena) {
    free(a.base);
    a.base = 0 as *u8;
    a.cursor = 0 as *u8;
    a.end = 0 as *u8;
}

/* ── 单测 ── */
fn main_jhyy() -> i32 {
    let a: Arena = Arena{base: 0 as *u8, cursor: 0 as *u8, end: 0 as *u8};
    arena_init(&a, 4096);

    /* alloc 4 sizes */
    let p1: *u8 = arena_alloc(&a, 8);
    let p2: *u8 = arena_alloc(&a, 16);
    let p3: *u8 = arena_alloc(&a, 32);
    let p4: *u8 = arena_alloc(&a, 64);

    /* verify monotonically increasing */
    if p1 >= p2 { return 1; }
    if p2 >= p3 { return 2; }
    if p3 >= p4 { return 3; }

    /* verify offsets */
    if (p2 - p1) != 8 { return 4; }
    if (p3 - p2) != 16 { return 5; }
    if (p4 - p3) != 32 { return 6; }

    /* reset and verify cursor back to base */
    arena_reset(&a);
    if a.cursor != a.base { return 7; }

    arena_free(&a);
    if a.base != 0 as *u8 { return 8; }

    0
}

### 6E.5 关键决策：alignment 处理

C 版用 `(uintptr_t)a->cur + (align-1)) & ~(align-1)` 做对齐。JHYY 没位运算 AND-not 简写。

**v0.6 决策**：翻译目标只暴露 arena_alloc(size)，对齐由调用方保证（调用方传已经是 8 倍数的 size）。这与原 arena.c 的"始终 8 字节对齐"语义保持兼容（如果所有调用方都按 8 倍数传 size）。

**v0.6.1+ 增强**：等 Stage 0 跑通后，加 `arena_alloc_aligned(a, size, align)` 用 `((cursor + align - 1) & mask)` 两步实现。**不在 v0.6 范围**。

## Potential Pitfalls

### 6A 切片
1. **16 字节布局对齐**：slice 是 `{*T, i64}` 16 字节 struct。Windows x64 上 ≤16 字节走寄存器返回，但**指针传递**仍可能走 sret（按调用约定细节）。需要实际生成 `.s` 检查 ABI 是否正确。
2. **数组字面量与切片字面量混淆**：`[1,2,3]` 是数组，`&[1,2,3]` 是 slice literal。parser 必须区分。
3. **slice as 函数参数**：v0.6 不要求支持 slice → 函数参数 / 返回。如果用户写 `fn foo(s: [*i32])`，codegen 应该能过（按 16 字节 struct pass-by-value）。需测试覆盖。

### 6B 命名空间
1. **mangled name 重复定义**：如果两个模块定义了同名函数，mangled 后不冲突（C 链接器认得）。但如果同模块内同名函数，仍是原来的错误（sema 报错）。
2. **import 循环**：现在的 `visited` 数组检测循环 import。引入 module 字段后，循环检测逻辑不变。
3. **transitive import 的 module 归属**：A import B, B import C，A 里 `b::c_func()` 应该是 `$b__c_func` 还是 `$c__c_func`？决策为**直接归属**：`c_func` 在 B 看来是 `c::c_func` 还是直接 `c_func`？**决策**：`c_func` 在 C 模块里 module=`"c"`，B 引用时是 `$c__c_func`（从 B 角度看是"跨模块限定"）；但 A 引用 B 的 `c_func`，写法是 `b::c_func()`，对应 `$b__c_func`？——这就要求 `b::c_func` 这种"传递性限定"语法。

**简化决策**：v0.6 不支持 `b::c_func`（传递性限定），只支持 `b::x`（B 自己的 x）。如果用户要 C 的函数，必须在 B 里 re-export，或在 A 里直接 `import c; c::x();`。

### 6C `as`
1. **指针 ↔ usize 零扩展 vs 符号扩展**：决策为**零扩展**（指针永远是非负）。
2. **`i32 as usize` vs `i32 as u64` vs `i32 as i64`**：QBE 上 `cast` 指令按 width 处理。`w` → `l` 用 `exts`/`extu`；`w` ↔ `w` 不需要 cast。
3. **`0 as *u8`**：常量 0 转指针是合法的（C 标准允许 NULL = 0）。v0.6 允许。

### 6D bug
1. **fallback 到 cwd 可能误命中**：cwd 下有同名文件时（`mylib.jhyy` 在 cwd 也有）会解析错。**修复**：仅在 main_path 目录找不到时 fallback。
2. **regress.py skip_solo 是 hack**：长期方案是 jhyy 支持 `--emit-asm` 不链接。v0.6 临时方案。

### 6E Stage 0
1. **alignment 算法**：见 6E.5，翻译时只暴露 alloc(size)，不暴露 alloc_aligned。
2. **Arena 用 `*Arena` 还是 `Arena`**：C 编译器把 Arena 作为不透明指针。JHYY 里 `*Arena` 更接近 C 的语义。
3. **struct 字面量 `Arena{base: 0, ...}`** 必须有所有字段。已支持（v0.2 struct 字面量）。
4. **指针算术 `a.cursor + size`**：JHYY 指针加整数已支持（v0.4 加 v0.5）。
