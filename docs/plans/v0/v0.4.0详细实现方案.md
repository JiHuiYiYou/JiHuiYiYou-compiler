## Sprint 4A: Struct Pass-by-Value

### 4A.1: Struct as Function Parameter

#### Problem
`cg_func` at `codegen.c:1027-1033` maps each JHYY parameter to a single QBE SSA temp via `copy`. Struct types need to be passed as pointers (`l` type in QBE) with the caller allocating a stack copy and the callee treating the pointer as the struct value.

#### Files: `codegen.c`

**Change 1: New function `cg_copy_struct`** (insert after `cg_emit_store`, around line 103)

Iterates over a struct's FieldDesc array and emits field-by-field load/store pairs. Recurses for nested struct fields.

```c
static void cg_copy_struct(CGContext *cg, Type *st, IRVal dst_addr, IRVal src_addr) {
    for (size_t i = 0; i < st->struct_type.nfields; i++) {
        size_t off = st->struct_type.fields[i].offset;
        Type *ft = st->struct_type.fields[i].type;
        IRVal src = src_addr, dst = dst_addr;
        if (off > 0) {
            src = ir_new_tmp(cg->ir, 'l');
            ir_emit_binary(cg->ir, src, "add", src_addr, ir_new_int((int64_t)off));
            dst = ir_new_tmp(cg->ir, 'l');
            ir_emit_binary(cg->ir, dst, "add", dst_addr, ir_new_int((int64_t)off));
        }
        if (ft->kind == KIND_STRUCT) {
            cg_copy_struct(cg, ft, dst, src);
        } else {
            IRVal fval = ir_new_tmp(cg->ir, qbe_type_of(ft));
            cg_emit_load(cg, fval, ft, src);
            cg_emit_store(cg, ft, fval, dst);
        }
    }
}
```

### 4A.2: Struct as Return Value

#### Problem
`ir_emit_ret` at `ir.c:123-128` emits a single scalar `ret`. Struct returns require the sret convention: caller allocates a return slot, passes a hidden pointer as the first argument, and the callee stores the result through it.

#### Files: `ir.h`, `ir.c`, `codegen.c`

**ir.h**: Add declaration (after line 56):
```c
void ir_emit_call_void(IRBuf *ir, const char *fn, IRVal *args, int n);
```

**ir.c**: Add implementation (after `ir_emit_call` at line 167):
```c
void ir_emit_call_void(IRBuf *ir, const char *fn, IRVal *args, int n) {
    ir_emit(ir, "    call $%s(", fn);
    for (int i = 0; i < n; i++) {
        if (i > 0) ir_emit(ir, ", ");
        if (args[i].kind == IRVAL_STR)
            ir_emit(ir, "%c %s", args[i].qbe_type, args[i].name);
        else
            ir_emit(ir, "%c %%t%d", args[i].qbe_type, args[i].id);
    }
    ir_emit(ir, ")\n");
}
```

**codegen.c: CGContext** (lines 8-19): Add sret fields:

### 4A.3: Struct Assignment

#### Problem
`cg_stmt NODE_ASSIGN` at `codegen.c:829-888` handles `NODE_DEREF`, `NODE_IDENT`, and `NODE_INDEX` targets, but none of these copy struct fields when the type is KIND_STRUCT.

#### Files: `codegen.c`

**Codegen.c: cg_stmt NODE_ASSIGN** (`codegen.c:838-845`)

In the NODE_IDENT branch, add struct type check:

```
NodeIdent *id = node_ident_data(d->target);
int is_stack = 0;
IRVal slot = cg_find_local(cg, id->sym, &is_stack);

if (d->target->type && d->target->type->kind == KIND_STRUCT) {
    // Struct copy: src is val (address), dst is slot (address or stack slot)
    IRVal dst_addr;
    if (is_stack) {
        dst_addr = slot;  // stack slot address
    } else {
        dst_addr = slot;  // is_stack=0, slot IS the address (from struct literal/param)
    }
    cg_copy_struct(cg, d->target->type, dst_addr, val);
} else if (is_stack) {
    cg_emit_store(cg, d->target->type, val, slot);
}
```

Note: `val` comes from `cg_expr(cg, d->value)` which for struct-typed expressions returns the address pointer (qbe_type='l').

## Sprint 4B: Multi-file Compilation

### 4B.1: Transitive Imports

#### Problem
`resolve_imports` at `main.c:154` skips transitive imports with `continue`. This prevents A->B->C import chains.

#### Files: `main.c`

**main.c: resolve_imports** (`main.c:88-174`)

Replace the single-pass approach with an iterative worklist algorithm:

```
static int resolve_imports(Node *module, const char *main_path, Arena *arena) {
    // Data structures (arena-allocated arrays as simple sets)
    const char **visited = arena_alloc(arena, 64 * sizeof(char*));
    int nvisited = 0;
    const char **in_progress = arena_alloc(arena, 64 * sizeof(char*));
    int nin_progress = 0;
    
    // Worklist: array of Sym* (module names to import)
    // Initialize from module's top-level import decls
    // ...
    
    // While worklist not empty:
    //   Pop import name
    //   Resolve path: dir/name.jhyy
    //   Check: already in visited? skip
    //   Check: in in_progress? cycle detected → error
    //   Add to in_progress
    //   Read and parse file
    //   For each decl in parsed module:
    //     If NODE_IMPORT_DECL and not visited: add to worklist
    //     Else: add to merged module decls
    //   Remove from in_progress, add to visited
```

Key differences from current code:
1. Worklist replaces single-pass iteration over module decls
2. Deduplication via visited set
3. Cycle detection via in_progress set
4. Transitive imports are NOT skipped - they're added to the worklist

The visited/in_progress arrays use simple linear search (O(n) with n < 64 in practice).

**main.c: resolve_imports error messages** (`main.c:126-127`)

Enhance error messages to include cycle paths when detected:
```
"%s:%d:%d: error: circular import detected: %s -> %s\n"
```

## Potential Pitfalls

1. **Struct return + nested struct**: sret for functions that return structs containing struct fields requires recursive cg_copy_struct. Ensure the recursion terminates.

2. **Struct param + field access compatibility**: After 4A.1, struct-typed IDs return addresses directly. All existing NODE_FIELD code already treats the base as an address (it adds offsets and loads). This should be compatible, but verify with the auto-deref path through pointers.

3. **Enum types**: Enum types (KIND_ENUM) have a similar problem to KIND_STRUCT for pass-by-value. However, the sprint scope is struct-only. Enum pass-by-value can be deferred.

4. **Array as param + struct**: Arrays are already passed by address (the address of the stack slot). Structs will now also pass by address. These should not conflict as the QBE types differ (array elements use element qbe_type; structs use `l`).

5. **QBE alloca in callee for sret**: The `%ret` parameter in sret is a pointer to the caller's stack. The callee must NOT alloca a new slot and copy - it should write directly through `%ret`. The `cg_copy_struct` to `cg->sret_slot` achieves this correctly.

6. **multiple return statements in sret function**: Each return in an sret function must copy to the same sret_slot, then ret void. Since the sret_slot is stored in CGContext, all return statements can reference it.
