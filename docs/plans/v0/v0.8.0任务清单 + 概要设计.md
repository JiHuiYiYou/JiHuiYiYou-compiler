### commit 2: 翻译 main.c → main.jhyy（#38, pending）

**目标**：`compiler/src/main.c` (523 行) → `compiler/src0/main.jhyy`

**挑战**：
1. **main.c 是 CLI 入口**（不是数据/算法模块）：`main(int argc, char **argv)` + `cmd_compile` / `cmd_build` / `cmd_run` / `cmd_dump`
2. **大量 FFI 依赖**：`fopen` / `fwrite` / `fread` / `fclose` / `system` / `exit` / `snprintf` / `malloc` / `free` / `sprintf_lld` / `jh_fputs_*`
3. **QBE 绝对路径调用**：`main.c:433` 写死 `C:/Users/liuzhen/Desktop/coding/JiHuiYiYou/qbe/qbe.exe`
4. **qbe 跟 gcc 命令构造**：sprintf 拼 cmd 字符串
5. **link 三个 .c 文件**：`runtime.c` + `jhyy_helpers.c` + 编译产出的 .s

**策略**：
- **不重翻译** v0.6 验证过的 `jhyy_helpers.c`（C 端唯一 FFI 桥，sprint 1 已就绪）
- **直接 link `main.jhyy → main.s` 进 self-hosted jhyy.exe**
- **重写 QBE/gcc 路径调用**：jhyy extern `system(cmd)` + jhyy 拼 cmd 字符串
- **MUST keep QBE 绝对路径**：v1.0.0 L1 计划第 134 行明确"qbe 二进制在 qbe/qbe.exe，不是 PATH 里，靠 jhyy main.c 写绝对路径调用"

**翻译顺序**（建议 3 个子 commit）：
- **2a**: jhyy extern 列表 + cmd_run（最简路径：编 + 跑 jhyy 程序）
- **2b**: cmd_compile / cmd_build（QBE 调用 + gcc 链接）
- **2c**: cmd_dump（AST 打印）

### commit 3: Stage 1 byte-equal 验证（#37, pending）

**目标**：
```bash
# 1. C 编译器编 jhyy → jhyy_0
gcc compiler/src/*.c -o jhyy_0 -I compiler/src

# 2. jhyy_0 编 main.jhyy → jhyy_1
jhyy_0 compile compiler/src0/main.jhyy -o jhyy_1

# 3. jhyy_0 vs jhyy_1 对相同 .jhyy 测试集产 byte-equal .il
mkdir ref
for test in regress_set; do
    jhyy_0 build $test -o ref/$test.il
    jhyy_1 build $test -o sh/$test.il
    diff -q ref/$test.il sh/$test.il   # 必须空
done

# 4. jhyy_1 跑 regress.py 全过
JHYY_CC=jhyy_1 python compiler/build/bin/regress.py
```

**风险**：
- 临时变量编号差异 → codegen.jhyy 强制 tmp 编号单调 + 固定 emit 顺序（sprint 4 翻译时已保证）
- 注释 / 空白差异 → 不允许（.il byte-equal 是硬性要求，per v1.0.0 L1 计划）
- data 段 emit 顺序 → 同样需保证稳定
