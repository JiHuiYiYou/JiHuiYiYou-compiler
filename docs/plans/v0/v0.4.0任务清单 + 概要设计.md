## Sprint 4A: Struct Pass-by-Value

### 4A.0: CGContext — Add sret fields

**File**: `codegen.c` lines 14-19 (CGContext struct)

Add two fields after `current_ret_type`:
```c
IRVal sret_slot;   /* hidden return slot pointer for struct returns */
int   has_sret;    /* 1 if function returns struct via sret */
```

### 4A.0b: Add `ir_emit_call_void`

**File**: `ir.h` (~line 56), `ir.c` (after `ir_emit_call`, line 167)

For sret calls: the call has no return value (void), because the result is written through the hidden pointer.

```c
void ir_emit_call_void(IRBuf *ir, const char *fn, IRVal *args, int n);
```

Implementation: same as `ir_emit_call` but emits `call $fn(...)` without `%tN =w ` prefix.

### 4A.1: `cg_copy_struct` helper

**File**: `codegen.c` (new function, after `cg_emit_store` ~line 103)

```c
static void cg_copy_struct(CGContext *cg, Type *st, IRVal dst_addr, IRVal src_addr) {
    // Iterate st->struct_type.fields[i]:
    //   compute src_off = src_addr + offset
    //   compute dst_off = dst_addr + offset
    //   load field value from src_off
    //   store field value to dst_off
    //   if field type is KIND_STRUCT, recurse
}
```

Key: uses `cg_emit_load` / `cg_emit_store` so sub-word types get correct width.

### 4A.2: Struct parameters — caller side (NODE_CALL)

**File**: `codegen.c` lines 364-379

In the argument loop, for each arg whose type is `KIND_STRUCT`:
1. Allocate a stack slot: `ir_emit_alloc(copy_slot, type_size(struct_type))`
2. Get src address: `cg_expr(cg, args[i])` → returns slot address (`'l'`)
3. Copy: `cg_copy_struct(cg, struct_type, copy_slot, src_addr)`
4. Set `args[i] = copy_slot` (qbe_type='l', the slot address)

### 4A.3: Struct parameters — callee side (cg_func)

**File**: `codegen.c` lines 1006-1033

**Header emit** (lines 1011-1016): For struct params, emit `l %name` instead of `qbe_type_of(pt) %name`.

**Param registration** (lines 1027-1033): For struct params:
```c
IRVal param_val = ir_new_tmp(ir, 'l');
ir_emit(ir, "    %%t%d =l copy %%%s\n", param_val.id, fd->params[i].sym->name);
cg_add_local(&cg, fd->params[i].sym, param_val, 0); // is_stack=0, value IS the pointer
```

### 4A.4: NODE_IDENT fix for struct types

**File**: `codegen.c` lines 207-218

When the variable type is `KIND_STRUCT` and `is_stack=1`: return `loc` directly (the slot address), do NOT load from it. Struct values are represented by their address.

```c
if (is_stack) {
    if (n->type && n->type->kind == KIND_STRUCT) {
        return loc;  // structs are always manipulated via address
    }
    // ... existing load logic for scalars
}
```

### 4A.5: NODE_LET with struct init

**File**: `codegen.c` lines 805-828

Add `is_struct` check alongside existing `is_array` check:
```c
int is_struct = (d->sym->type && d->sym->type->kind == KIND_STRUCT);
if (d->is_mutable || is_array || is_struct) {
```

For struct literal init (similar to array literal optimization): use init's slot directly. Otherwise: alloc slot, copy struct via `cg_copy_struct`.

### 4A.6: Struct assignment (NODE_ASSIGN)

**File**: `codegen.c` lines 829-888

In the `NODE_IDENT` branch, when target type is `KIND_STRUCT`:
1. Get src addr: `cg_expr(cg, d->value)` → returns address
2. Get dst addr: `cg_find_local(cg, id->sym, &is_stack)` → returns address
3. `cg_copy_struct(cg, struct_type, dst_addr, src_addr)`

### 4A.7: Struct return — callee (NODE_RETURN)

**File**: `codegen.c` lines 890-899

When returning from a function with `has_sret`:
```c
if (cg->has_sret && dr->expr) {
    IRVal src = cg_expr(cg, dr->expr);
    cg_copy_struct(cg, cg->current_ret_type, cg->sret_slot, src);
    IRVal v = {0};
    ir_emit_ret(cg->ir, v);  // bare ret
}
```

### 4A.8: Struct return — callee header (cg_func)

**File**: `codegen.c` lines 999-1033

When `ret_type` is `KIND_STRUCT`:
- Emit: `export function $name(l %ret, ...)` — first param is hidden sret pointer
- Register sret in CGContext: `cg.has_sret = 1; cg.sret_slot = ret_copy;`
- Skip `ret_qt` for result (set to 0 = void)

The implicit return at end (line 1037-1051): when `has_sret`, emit bare `ret`, do not emit `ret %body_val`.

### 4A.9: Struct return — caller (NODE_CALL)

**File**: `codegen.c` lines 364-379

When the function return type is `KIND_STRUCT`:
1. Allocate return slot: `ir_emit_alloc(ret_slot, type_size(ret_type))`
2. Prepend `ret_slot` as first argument (before all user args)
3. Use `ir_emit_call_void` (no return value)
4. Return `ret_slot` as the expression result

Detection: check `n->type && n->type->kind == KIND_STRUCT`.

## Sprint 4B: Multi-file Compilation

### 4B.1: Transitive import resolution

**File**: `main.c` lines 88-174 (`resolve_imports`)

Rewrite with iterative work-list algorithm:

```
visited_paths = {}     // modules already processed
in_progress = {}       // modules currently resolving (for cycle detection)

work_list = main_file's NODE_IMPORT_DECL entries

while work_list not empty:
    pop (mod_name, source_loc)
    mod_path = dir + "/" + mod_name + ".jhyy"
    
    if mod_path in in_progress:
        emit error "circular import: <chain>"
        continue
    
    if mod_path in visited:
        continue  // already processed
    
    in_progress.add(mod_path)
    
    read mod_path
    parse into arena (same arena as main)
    
    for each decl in parsed module:
        if NODE_IMPORT_DECL: push to work_list
        else: add to merged decls
    
    visited.add(mod_path)
    in_progress.remove(mod_path)

replace module->decls with merged decls
```

### 4B.2: Multi-file CLI input

**File**: `main.c` — `cmd_compile` and `compile`

Change `cmd_compile` to accept multiple input files:
```
jhyy compile a.jhyy b.jhyy -o output
```

Implementation:
1. Parse argv: collect all non-flag args as input files
2. First input file is the "main" module
3. Parse each input file into a separate AST module
4. Merge all non-import decls from all modules
5. Run resolve_imports on the merged module
6. Run sema + codegen as before

### 4B.3: Fix cmd_build

**File**: `main.c` — `cmd_build`

Insert `resolve_imports` call between parser_parse and sema_check.

## Risk Notes

1. **Struct field recursion**: `cg_copy_struct` must handle nested structs. Check field type kind and recurse.
2. **sret + existing return logic**: The `body_returns` macro and implicit return must all respect `has_sret`.
3. **Import arena sharing**: All parsed modules must use the same arena (already the case), or sym pointers will dangle.
4. **Import cycle error messages**: Should report the full cycle chain, not just "circular import".
5. **Enum pass-by-value**: NOT in scope. Enums already work because they're scalar-sized.
