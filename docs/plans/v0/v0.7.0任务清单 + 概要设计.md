# v0.7.0 任务清单 + 概要设计

## Context

v0.6.x（v0.6.3 / v0.6.4 / v0.6.5）走小 patch 流修了 #9 / #10 / #2 三个 codegen / sema bug，phase-2 自举 v1.0.0 sprint 3 已推进到 parser.jhyy parse_decl（sprint 3 commit 2，2026-06-25）。v1.0.0 翻译继续。

但自举 v1.0.0 sprint 3 实测碰到两个**语言层缺口**，光靠 C 端 codegen patch 解决不了：

1. **match 缺穷尽性检查**：`match opt { Option::Some(v) => v, }`（漏 None arm）当前**编译通过、运行时崩**。sprint 3+ 翻译 parser/sema/ir/codegen 时，switch over NodeKind / TokenKind 大量出现，每次都得手写通配符 `_ =>` 兜底——既脏又容易漏。自举内部会高频写 match-on-enum，不补穷尽性检查会反复踩坑。
2. **缺顶层 `const` 数组**：opcode table、ASCII table、TokenKind→string 映射表这类数据，jhyy 端只能用 growable arena_alloc + 运行时初始化（或硬编码在 C 端）。**不是 codegen bug，是语言层缺特性**——C 编译器 v0.6 无 `const NAME: [T; N] = [...]` 顶层声明。

### 为何不开 v0.6.6 patch 而开 v0.7.0

两个特性都涉及**新增语言层语法**（match 检查 / const 关键字 + 顶层声明），不是 codegen 局部 fix。按 `phase-1-v0.2.x-to-v1.0.0.md` 的"严格按依赖顺序推进"原则，**新语法 = 新版本号**（v0.6.x 保留为 bug 修复 patch）。

v0.7.0 是 v0.6.5 patch 之后的**第一个特性 minor version**。

## 目标

### 1. enum first-class

**现状**（v0.6）：
- `type Option = enum { Some(i32), None };` 定义 OK
- `let x = Option::Some(42); let y = Option::None;` 实例化 OK
- `match opt { Option::Some(v) => v, Option::None => 0 }` 模式匹配 OK
- `Option::Some as i32` 取 tag OK（已经在 codegen 走）

**缺**：
- **穷尽性检查缺失**：match on enum 没覆盖所有 variant 时，**不报错**，运行时崩
- **`_` 兜底过宽**：所有 match 都得加 `_ => unreachable() / panic()` 才能不漏

**v0.7 加**：
- sema `NODE_MATCH` case：match expr 类型是 enum 时，**穷尽性检查**——所有 variant 必须有 arm；缺则 `sema_error` "non-exhaustive match: missing variant 'None'"
- `_` 仍可作通配符，但**仅当所有 variant 都覆盖后**允许（即 `_` 不是 variant 覆盖的替代，是"已覆盖后再加 default"）

### 2. const struct array

**现状**（v0.6）：
- 顶层声明只支持 `fn` / `type` / `extern fn` / `import`（parser.c `parse_decl` 第 530 行）
- 无 `const NAME: [T; N] = [...]`
- 数组只能运行时初始化（growable arena_alloc）

**v0.7 加**：
- 新 `const` 顶层声明：`const NAME: [T; N] = [expr, expr, ...];`（T 限定为原语 / struct，不能含 enum / 指针运行时分配）
- 顶层 const 表达式必须是**编译时可求值**字面量（i32 / f64 / bool / struct 字面量 / 其他 const 引用）
- codegen emit 到 QBE `data $name = { b ... }` 段（zero-cost，运行时直接 load 地址）
- 访问：`let arr = ASCII; let c = arr[65];`（自动 decay 成数组变量）

## Sprint 7A: enum 穷尽性检查

**目标**：`match opt { Option::Some(v) => v }`（漏 None）必须编译错误。

### 7A.1 variant 列表获取

`symtab` 已经维护 `SYM_VARIANT` 节点（在 type enum 定义时插入）。新增 API：

```c
// symtab.h
int sym_enum_variants(SymTable *t, const char *enum_name, 
                      Sym **out_variants, int max);
```

返回 enum 类型的所有 variant sym（按定义顺序）。

### 7A.2 匹配分析

sema `case NODE_MATCH`：
1. 取 match expr 类型 `T`
2. 如果 `T->kind == KIND_ENUM`：
   - 调 `sym_enum_variants` 取所有 variant
   - 遍历 match arms，对每个 arm 的 pattern：
     - `Enum::Variant(x)` → 标记该 variant 已覆盖
     - `Variant(x)` → 同上（v0.7 允许短名匹配 enum variant）
     - `_` → 所有 variant 标记覆盖（但是**最后兜底**——前面 arm 不许再出现 variant pattern）
   - 检查未覆盖 variant：每个未覆盖的 variant 报 `sema_error "non-exhaustive match: missing variant 'None'"`（sprint 7A 加 `-Werror` 严格化）
3. `T->kind != KIND_ENUM`：现有逻辑不变（int / string / wildcard 模式自由匹配）

### 7A.3 短名 variant pattern

`match opt { Some(v) => v, None => 0 }` 替代 `Option::Some(v) => v, Option::None => 0`：
- parser `parse_pattern` 遇到 `IDENT` 时，如果当前 scope 里有同名的 SYM_VARIANT，允许短名
- 短名 match 进 codegen 时补全 `EnumName::Variant` 形式（通过 symtab lookup variant 找到所属 enum）

### 7A.4 兼容

- 现有 47 个 regress 测试都加 `_ => unreachable()` 兜底 arm——穷尽性检查**不会破坏**（已有兜底 = 全部 variant 视为已覆盖）
- 新加 negative test：`match opt { Option::Some(v) => v }`（漏 None）必须编译错误

## Sprint 7B: const struct array

**目标**：`const ASCII: [u8; 128] = [0, 1, 2, ...];` 顶层声明，运行时零成本访问。

### 7B.1 语法

```jhyy
// 顶层（与 fn / type / extern / import 平级）
const ASCII_LOWER: [u8; 26] = [
    97, 98, 99, 100, 101, 102, 103, 104, 105, 106, 107, 108, 109,
    110, 111, 112, 113, 114, 115, 116, 117, 118, 119, 120, 121, 122,
];

type Opcode = struct { name: *u8, code: i32 };
const OPCODES: [Opcode; 3] = [
    Opcode { name: "add", code: 1 },
    Opcode { name: "sub", code: 2 },
    Opcode { name: "mul", code: 3 },
];
```

**约束**：
- T 限定为 **i32 / i64 / u8 / f32 / f64 / bool / struct**（不能 enum / 不能含指针 / 不能含 slice）
- 长度 N 必须是**字面量整数常量**
- 初值必须是**字面量表达式**或**其他 const 引用**（不能调函数、不能含变量）

### 7B.2 parser

新增 `parse_const_decl`：
- `const` token（lexer 加 TOKEN_CONST）
- `NAME` ident
- `: [T; N]` 类型注解
- `= [...]` 数组字面量（必须是 `NODE_ARRAY_LIT`，所有 elem 走 const expression check）
- `;` 结尾

parser 调用 `symtab_insert(p->global_scope, name, SYM_CONST, type, false, 0)`，`is_mutable=false` 标记不可重赋值。

### 7B.3 sema

`case NODE_CONST_DECL`：
- 校验初值所有 elem 是 const 表达式（literal / struct lit 字面量 / 其他 SYM_CONST 引用）
- 校验 elem 类型匹配 `T`
- 校验 `N` 是字面量 i32
- 校验 T 不含 enum / 指针 / slice

const 引用传播：访问 `let x = ASCII[65];` 时 sema 查 `SYM_CONST`，返回底层 type。

### 7B.4 codegen

`case NODE_CONST_DECL`：
- 调 `ir_emit_data_array(ir, name, elem_size, n_elems, init_bytes)`
- 对每个 elem 调用 `cg_const_init_value` 取字面量（i32 / f64 / struct 字面量），按 QBE `data` 段格式 emit
- 注册到 codegen 的 global symbol 表，让后续 load `ASCII` 变成 load `data $ASCII` 地址

`ir_emit_data_array`：
```c
void ir_emit_data_array(IRBuf *ir, const char *name, size_t elem_size, 
                        size_t n_elems, const char *init_bytes) {
    ir_emit_data(ir, "data $%s = { ", name);
    for (size_t i = 0; i < n_elems; i++) {
        // 每个 elem 按 elem_size emit b / w / l 字面量
        // ...
    }
    ir_emit_data(ir, " }\n");
}
```

`cg_load_const_array`（新）：
```c
IRVal cg_load_const_array(CGContext *cg, Sym *sym) {
    IRVal addr = ir_new_tmp(cg->ir, 'l');
    ir_emit(cg->ir, "    %%t%d =l addr $%s\n", addr.id, sym->name);
    return addr;  // 返回数组基址，后续 index 跟普通数组一样
}
```

### 7B.5 兼容

- 现有 47 个 regress 测试**不涉及 const 顶层声明**——0 改动
- 新加 positive test：`const X: [i32; 3] = [1, 2, 3];` 顶层声明 + `let y = X[0];` 访问，验证 QBE `data $X = { w 1, w 2, w 3 }` + `addr $X` load

## v0.8+ 候选：自举过程发现 codegen bug

v1.0.0 phase-2 自举 sprint 4（翻译 `compiler/src/ir.c` → `compiler/src0/ir.jhyy`，2026-06-30）实测碰到 **4 个 C 端 codegen bug**。当前**全部用 jhyy 端 workaround 绕开**（jhyy 源码里有 `// v0 codegen bug:` 注释作为 TODO），stage 1 IL 字节对齐（`_driver_ir.jhyy` 23/23 通过，exit 42）达成。

**v0.8 计划：一次性集中修完这 4 个 bug**，依据是：(1) 每个 bug 走 codegen 全管线，单独修各 ~1 sprint；(2) 同一文件多次趟 codegen 不如一次性修干净；(3) v1.0.0 自举验证已暴露所有真实触发路径，根因定位容易。

### Bug A: nested struct 字段地址 LEA 错误

- **触发**：`&((*ptr).field)` ——指针解引用后取 struct 字段地址
- **错误现象**：codegen emit `loadw`（或对应字段类型）而不是 LEA（`add %ptr, offset`）；上层若需要 `*T` 指针，QBE 类型不匹配
- **根因猜测**：codegen 看到 `field` 是 i32/long 就按值 load，没识别外层是要地址
- **jhyy workaround**：把 inline struct 改成 `*Pointer` ——`type IRBuf = struct { ..., sb: *StringBuilder, ... }` 改 `sb: *StringBuilder`，StringBuilder 从 arena 分配
- **v1.0.0 位置**：`compiler/src0/ir.jhyy` `type IRBuf` 定义、`fn ir_init`（line ~80）
- **修完后影响**：jhyy 端可恢复 inline struct 写法（更直观），但 QBE 输出不变

### Bug B: nested if-else phi predecessor 错

- **触发**：`if A { if B {...} else {...} } else if C {...} else {...}` 这种 3-way dispatch，merge block 的 phi 前驱列表写错
- **错误现象**：QBE 报 `predecessors not matched in phi %tXXX`
- **根因猜测**：codegen 写外层 merge block 的 phi predecessors 时，列出的是内层 if 的 **start block**（如 `@then2`）而不是 end block（如 `@merge2`）
- **jhyy workaround**：拆成 dispatcher + N helper 函数，每层函数只 1 级 `if`，helper 用 early return 避免嵌套
- **v1.0.0 位置**：`compiler/src0/ir.jhyy` `ir_emit_binary` → 拆成 `ir_emit_binary`（dispatcher）+ `_bin_int` + `_bin_tmp` 三个函数
- **修完后影响**：sprint 后续翻译（codegen.jhyy / sema.jhyy）的类似嵌套 if-else 不再需要拆函数

### Bug C: `*u8` deref emit loadub 而非 loadl

- **触发**：`*((ptr) as *u8)` 解引用 8 字节指针值（实际目的是拿 8 字节 long 值再 cast）
- **错误现象**：codegen 看到目标类型是 `*u8` 就走 `loadub`（1 字节 unsigned byte load），应该走 `loadl`（8 字节 long load）
- **根因猜测**：codegen 走 `*T` 路径时按 T 的元素大小决定 load 宽度，没考虑上层 cast 后的使用方式
- **jhyy workaround**：用 `*i64` deref + cast ——`let raw_i64 = *((ptr) as *i64); let p = raw_i64 as *u8;`
- **v1.0.0 位置**：`compiler/src0/ir.jhyy` `ir_emit_func_header` 取 `params_name[i]`（line ~764）
- **修完后影响**：`params_name: *u8`（pointer-to-pointer）直接 deref 即可

### Bug D: `&local_var` 漏 store

- **触发**：`fn foo() { let b: u8 = 0; emit(&b); }` 取 local stack var 地址
- **错误现象**：codegen 给 `&b` alloc 栈空间（`%t =l alloc4 1`）但**没 store** `b` 的值；后续 `storeb %t0, %tADDR` 时 `%t0` 未初始化（实际是上一个 register 的残留）
- **根因猜测**：codegen 看到 `&b` 就 alloc 但不 emit 初始 store；调用方拿到地址后立即写自己的值，但 codegen 内部用 tmp 寄存器传值会丢失
- **jhyy workaround**：用 `arena_alloc(4) + *i32` 写低字节 ——`let buf = arena_alloc(arena, 4); let p = buf as *i32; *p = val; emit(buf);`
- **v1.0.0 位置**：`compiler/src0/ir.jhyy` `data_putc`（line ~683）
- **修完后影响**：临时 char 缓冲可直接 `let b: u8 = val; emit(&b);` 不再需要 arena_alloc 绕开

## 风险

- **7A 短名 variant pattern**：可能跟现有 variable ident 冲突（`match opt { Some(v) => ... }` 里的 `Some` 是 enum variant 还是局部变量？）。sema 走"先查 SYM_VARIANT，无则查 SYM_VAR"——遇到歧义报错。
- **7B struct 字面量在 const 初值**：struct 字段如果是 `*u8`（name 字段）需要 const 字符串。**v0.7 不支持 const 字符串字面量**——只能是 i32 / 数字 / bool 字段。struct 字段限定为**纯字面量类型**。
- **QBE data 段大小限制**：QBE 单 data 段有 64KB 限制。v0.7 不考虑——typical OPCODES table < 1KB。超大表走 `addr $giant + offset` 拆分（v0.8+）。
