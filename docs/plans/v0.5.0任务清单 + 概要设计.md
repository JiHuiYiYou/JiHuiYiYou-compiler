## Sprint 5A: 全部 Bug 修复

### 5A.1 修复 `*ptr = *ptr - expr` sema 类型推断

**症状**: RHS 上对 `*ptr` 求值时，sema 的 `infer_type` 走到 `NODE_BINARY` 时把 LHS 当作"待赋值的引用"，RHS 上的 `*ptr` 推不出正确类型。
**根因**: `sema.c` 的二元运算子节点递归中，`*ptr` 在 RHS 上需要先被识别为 deref 表达式 (与 LHS 区分)。
**修复**: 在 `infer_type` 处理 `NODE_BINARY` 时，确保两侧子表达式独立走完类型推断（包括 `NODE_DEREF` 路径），然后再做 `lhs_type = rhs_type` 的赋值兼容性检查。
**回归测试**: `compiler/tests/examples/ptr_self_assign.jhyy`，`let mut x: i32 = 100; let p = &mut x; *p = *p - 30; exit(*p)` → EXIT:70。

### 5A.2 修复嵌套 if/else if/else phi 前驱块标签

**症状**: `if/else if/else` 链的 phi 节点前驱块 (predecessor) 标签指向了 fallthrough 块而非真正的条件分支结束块，导致 SSA 不合法，运行时读到未初始化变量。
**根因**: `codegen.c` 的 `cg_if` 在构造 else-if 链时，复用了上一级的"中间块"作为下一级 if 的"then 块"，但 phi 节点还在用更早的块作为前驱。
**修复**: 引入 trampoline 块策略 — 每个 else-if 都新建自己的 cond/end 块，最终统一汇入 join 块。
**回归测试**: `compiler/tests/examples/nested_if.jhyy`，构造 ≥3 层 `if/else if/else` 并让所有分支都被走到。

### 5A.3 修复 if/else void 分支 phi 填充类型错误

**症状**: void 返回函数里的 if/else 链在 else 分支没有被实际进入时，phi 节点被填充成 `%t0` (默认 word) 而非 void，QBE 报错。
**修复**: phi 发射时增加 void 类型判断，void 走单独分支不填值。

### 5A.4 浮点算术 codegen (f32/f64)

**现状**: f32/f64 字面量 codegen 完成 (v0.3.0)，但 `+ - * /` 仍用整数 opcode，结果是 0。
**修复**: `codegen.c` 的 `cg_binary` 对 `KIND_FLOAT` (f32) 和 `KIND_DOUBLE` (f64) 类型使用 'add' / 'sub' / 'mul' / 'div'，并按 qbe_type ('s' / 'd') 发射。
**回归测试**: `float_arith.jhyy`，`let a: f64 = 1.5; let b: f64 = 2.5; let c = a + b * 2.0; exit(c as i32)` → EXIT:6。

### 5A.5 break/continue 支持

**现状**: 关键字未识别，遇到就 syntax error。
**修复**:
1. `lexer.h/c`: 新增 `TK_BREAK` / `TK_CONTINUE`
2. `parser.h/c`: 新增 `NODE_BREAK` / `NODE_CONTINUE`，放在循环体内合法
3. `sema.c`: 校验 break/continue 只在 while/for 内出现
4. `codegen.c`: while 循环用 `@loop_start` / `@loop_end` QBE 块标签，break 跳 `@loop_end`，continue 跳 `@loop_start`

**回归测试**: `break_continue.jhyy`，for 循环 + break + continue 验证。

### 5A.6 大整数溢出行为定义

**现状**: 未定义。wrap、trap、panic 都不确定。
**决策** (写入 lang-spec): 整数算术采用二进制补码 wrap 语义 (与 LLVM/GCC 默认一致)，无运行时检查。溢出检测留给上层。
**回归测试**: `overflow.jhyy`，`let mut x: u8 = 255; x = x + 1; exit(x as i32)` → EXIT:0。

### 5A.7 清理临时文件

- 删除 `compiler/tests/examples/_test_*.jhyy` (15 个)
- 删除 `compiler/build/_*.txt` (历史调试日志)
- 更新 `.gitignore` 排除新的临时模式

## Sprint 5B: ABI 白皮书 v1.0.0

**输出**: `docs/abis/jhyy-abi-v1.0.0.md`

**章节结构**:

1. **类型布局**: primitive (i8~u64 / f32 / f64 / bool), pointer, array `[T; N]`, struct, enum (tag + payload), slice `[*]T` (规划中)
2. **sizeof / alignof 表**: 每个类型的大小和对齐值
3. **函数调用约定**: 参数顺序 (QBE 原生左→右), 返回值 (scalar vs sret), struct pass-by-value (栈拷贝 + 指针传递)
4. **QBE IL 指令映射速查表**: JHYY 表达式 → QBE IL 指令 (load / store / binary / call / ret / phi)
5. **FFI 边界规则**: extern fn 的名字修饰 (无修饰), C ABI 兼容性 (Windows x64 PE), struct FFI 限制
6. **名称修饰方案**: JHYY 内部符号 (`main_jhyy` 不修饰), extern 符号 (C 原样)
7. **内存模型**: 栈 (QBE 分配), 静态数据 (QBE `data` 段 → `.rodata`), Arena (FFI runtime 提供)
8. **版本兼容性承诺**: v0.5.0 → v1.0.0 保证 ABI 稳定
9. **Windows x64 PE 特定行为**: `printf` 中文 UTF-8 输出 (SetConsoleOutputCP), path separator (`/` only)
10. **C ABI 互操作**: 哪些类型可以直接传给 C 库 (i32/u32/i64, *u8), 哪些需要 adapter (struct, enum)

**验收**: 与 C 编译器的 struct layout 对齐验证表 (手算 + 测试)。

## Sprint 5C: Claude Code MCP 服务

**技术栈**: Python FastMCP (用户指定)
**目录**: `mcp-jhyy/`
**入口**: `mcp-jhyy/server.py`

### 5C.1 骨架与注册

```python
# mcp-jhyy/server.py
from fastmcp import FastMCP
mcp = FastMCP("jhyy")

@mcp.tool
def jhyy_compile(file: str, output: str = None) -> dict:
    ...

if __name__ == "__main__":
    mcp.run()
```

注册到 `~/.claude.json` 的 `mcpServers.jhyy`：
```json
"jhyy": {
  "command": "python",
  "args": ["C:/Users/liuzhen/Desktop/coding/JiHuiYiYou/mcp-jhyy/server.py"]
}
```

### 5C.2 核心工具

| 工具 | 输入 | 输出 |
|------|------|------|
| `jhyy_compile` | file: str, output?: str | `{success, exit_code, stdout, stderr, diagnostics[]}` |
| `jhyy_run` | file: str | `{exit_code, stdout, stderr}` |
| `jhyy_check` | file: str | `{success, diagnostics[]}` (不生成代码，只 parse + sema) |

`diagnostics` 格式: `{file, line, col, severity, message}`。

| `jhyy_format` | 验证语法正确性（复用 jhyy_check 的 parser 阶段） |

## Risk Notes

1. **浮点算术 QBE opcode**: 必须按 qbe_type 选 'add'/'sub'/'mul'/'div'，不能复用整数路径。需要 audit 所有二元运算类型分发点。
2. **break/continue 与 sret 嵌套**: break 在 sret 函数中需要先复制到 sret_slot 再跳 (实际是直接跳，sret_slot 此时已写)。需要在 codegen 验证。
3. **MCP 服务 Windows 路径**: subprocess 调用 jhyy.exe 时路径分隔符必须用 `/`。
4. **MCP 服务中文输出**: jhyy_run 的 stdout 可能含中文，需要 MCP 端不做编码转换直接转发。
5. **ABI 白皮书稳定承诺**: v1.0.0 是 self-hosting 前最后承诺版本，自举后 ABI 改动必须走 v2.x。
